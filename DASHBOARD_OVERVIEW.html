2. System Architecture

Backend (FastAPI + Python)

app/main.py — FastAPI app, all 12 routers registered under /api/v1, CORS, lifespan startup
app/investments/ — Investment CRUD (name, type, series, description, notes)
app/securities/ — Securities per investment (round, size, PPS, dates)
app/documents/ — File upload (50MB limit), download, view, delete; stored in /uploads/
app/financial_parsing/ — Chunked PDF parsing with Claude, statement extraction, review/lock workflow, ratio computation, period comparison, trend aggregation
app/valuations/ — Valuation records with methodology, multiple, EV, PPS; Excel export
app/ai_insights/ — SSE streaming endpoint, compact context assembly (<1500 tokens), Claude API streaming
app/comparables/ — Comp sets, company profiles, FMP market data, benchmark lookup
app/financial_tracker/ — Period receipt grid (which investment has Q1 IS, Q2 BS, etc.)
app/migrations.py — Idempotent ALTER TABLE for new columns; dual-dialect (SQLite PRAGMA / Postgres information_schema)

Frontend (React + Vite + Tailwind)

src/api.js — Centralized API client; all 40+ endpoints wrapped in named methods
App.jsx — Root: investment list sidebar + InvestmentPanel main content
InvestmentPanel.jsx — Tab shell: Overview / Financials / Valuations / Comparables / AI Insights
FinancialStatements.jsx — Full-screen modal: parse progress, line item table, period label picker, review/lock workflow
FinancialDataView.jsx — Financials tab: Ratios&KPIs / Statement View / Period Comparison / Period Changes / Key Trends
RatioKPICards.jsx — 9 computed ratio cards with delta badges; refreshes on statements-approved event
AIInsightsPanel.jsx — Chat UI with SSE stream reading, scope toggle (single / all), react-markdown rendering
UploadModal.jsx — Drag-and-drop upload, 50MB guard, auto-parse PDFs after upload
PriorityQueue.jsx — Orange banner widget: lists documents needing action, "Parse Now" button per item
ValuationPanel.jsx — Valuation form + history chart (Recharts), Excel export

4. AI Parsing Pipeline — Document → Statements

Upload PDF — POST /documents/bulk → stored at uploads/{investment_id}/{uuid}.pdf
↓
Auto-triggered immediately after upload
Parse Job Created — status: pending → splits PDF into chunks of 25 pages (overlap: 5)
↓
Background processing — frontend polls /financials/status every 2s
Claude Extraction (claude-sonnet-4-20250514) — each chunk → structured JSON with IS / BS / CF line items, categories, is_total flags, currency, unit
↓
status: completed — progress bar fills to 100%
Review in FinancialStatements.jsx — view all 3 statement types, edit individual line items, click to edit values/labels inline
↓
Period label picker: dropdown Q1–Q4 / FY / FY Audited + custom
Map to Investment — POST /map-investment sets investment_id + fiscal_period_label → statement becomes available for ratio computation and AI context
↓
"Approve" dispatches statements-approved event → RatioKPICards auto-refreshes
Normalize Labels — optional: maps raw extracted labels → canonical categories (revenue, gross_profit, etc.) for cross-period alignment

5. Ratio Computation — 9 KPI Cards

Income Statement Ratios

Gross Margin = Gross Profit / Revenue × 100 — shows delta vs prior IS period
EBITDA Margin = (Op. Income + D&A) / Revenue × 100 — uses EBIT as proxy if D&A unavailable
Net Margin = Net Income / Revenue × 100 — delta vs prior period
Revenue Growth YoY — true YoY: same fiscal quarter, prior year; requires period label to parse (Q1 FY2025 vs Q1 FY2024)

Balance Sheet & Cash Flow Ratios

Current Ratio = Current Assets / Current Liabilities — liquidity check
Quick Ratio = (Cash + ST Investments + AR) / Current Liabilities — stricter liquidity
Debt / Equity = (ST Debt + LT Debt) / Equity — leverage; delta red = good
Burn Rate = |OCF| / period months — shows "Profitable" if OCF ≥ 0; monthly
Runway = Ending Cash / Monthly Burn — only shown when burning; "Profitable" if OCF ≥ 0

6. AI Insights — Context & Streaming

Context Assembly (service.py)

Investment header: name, asset type, series
Securities: round, date, size, cost basis per share
Financials: pipe-delimited compact rows (cat|value|period), is_total only, max 6 periods — sorted approved > reviewed > pending
Ratios: all 9 computed ratios in compact format (GrossMargin:42.3%@Q3FY25 Δ=+2.1pp)
Valuations: last 6 records (date, methodology, multiple, PPS, EV)
Token budget: <1,500 tokens single investment · <2,000 tokens all-investments (IS totals only) · System prompt <150 tokens

Streaming Architecture

POST /api/v1/ai-insights/query → FastAPI StreamingResponse (text/event-stream)
Each chunk: data: {"text":"..."}\n\n — terminated by data: [DONE]
Frontend uses native fetch + ReadableStream + TextDecoder — NOT the api.js JSON client
Accumulated text shown live with cursor blink; finalized as react-markdown on [DONE]
Conversation history passed client-side (session only — no server persistence)
Context injected into first user message as <financial_data> block; subsequent turns are plain text
nginx: proxy_buffering off + X-Accel-Buffering: no for SSE pass-through

7. Key API Endpoints Reference

Method Endpoint What it does
POST/api/v1/investments/Create a new investment company record
POST/api/v1/investments/{id}/securities/Add a security (round, size, PPS, date) to an investment
POST/api/v1/investments/{id}/documents/bulkUpload 1+ documents; returns array of DocumentResponse
POST/api/v1/investments/{id}/documents/{doc}/financials/parseQueue a PDF for AI parsing (async background job)
GET/api/v1/investments/{id}/documents/{doc}/financials/statusPoll parse job status (pending/processing/completed/failed)
GET/api/v1/investments/{id}/documents/{doc}/financials/Get all extracted statements + parse job for a document
POST/api/v1/financials/statements/{id}/reviewSet review_status (pending/reviewed/approved)
POST/api/v1/financials/statements/{id}/map-investmentLink statement to investment + set fiscal_period_label
PATCH/api/v1/financials/line-items/{id}Edit a line item value or label (creates user_modified record)
GET/api/v1/dashboard/financials/{id}Period-aligned comparison dataset for all statement types
GET/api/v1/dashboard/financials/{id}/ratiosCompute all 9 financial ratios from latest available statements
GET/api/v1/dashboard/financials/{id}/changesPeriod-over-period absolute and % change for every line item
GET/api/v1/dashboard/financial-trends/{id}Key metric time series for mini bar chart sparklines
POST/api/v1/investments/{id}/valuations/Record a new valuation (methodology, multiple, EV, PPS)
GET/api/v1/investments/{id}/valuations/exportDownload valuation history as Excel
POST/api/v1/ai-insights/querySSE streaming: natural language Q&A over financial context
GET/api/v1/priority-queue/investments/{id}List documents needing attention (unparsed, failed, low confidence)
GET/api/v1/workflow/investments/{id}Document workflow summary: counts by stage (not_parsed → approved)

10. Deployment

Docker Compose (Production)

db — Postgres 16, pgdata named volume, healthcheck
backend — Python 3.12, depends on db healthy, uploads volume mounted
frontend — nginx multi-stage build; proxies /api → backend; SPA fallback; SSE buffering off
DATABASE_URL — set via .env (FINANCE_DATABASE_URL=postgresql://...)
Startup: docker compose up --build

Local Development

Backend: uvicorn app.main:app --reload — SQLite by default
Frontend: cd frontend && npm run dev — Vite on :5173, proxies /api → :8000
Env: copy .env.example → .env, set FINANCE_ANTHROPIC_API_KEY
DB init: automatic on first start (create_all + migrations)
Dual-dialect: migrations.py works with both SQLite and Postgres
